Error Codes
Other > Error Codes


Bug 101

Security is compromised on FAT(32) because of the lack of permissions. To fix this, simply convert your drive to NTFS.

Bug 102

The source of SD-Security could possibly be modified, so always check that the SDS exe is signed with the SD-Storage certificate.

Bug 103

SD-Security has to be launched in the same directory as the exe, to retain the configuration. plugins. sessions, data vault etc.

Bug 104

If the drive is NTFS and you use SDS on one PC, then use it on another PC, then SDS will display an error with the settings file

DEV 2 Bug

This security flaw, allows the user to inject code into SD-Security by using ampersands, quotes and other special characters in input fields.

DEV 3 Bug

This security flaw, allows the user to inject code into SD-Security by using ampersands, quotes and other special characters by changing system variables, eg. %time%, %date%, %cd% etc.

DEV 5 Bug

The time & date API displays the date/time wrong. This is because of an unsupported time format/zone.
